## 早期リターン
早期リターン（Early Return）とは、
条件を満たしたら、その場でメソッドを終了する書き方
実はRailsでは早期リターンをガード節（Guard Clause）と呼ぶことも多いです。

## 判断基準

「この条件がダメなら、この先の処理を続けても意味がないか？」

YES → 早期リターンを使うのが自然です。
NO → 通常の if 文で処理を分岐させる方が分かりやすいこともあります。

Railsでは「異常系（エラー・権限なし・データなし）」を最初に return で処理し、最後に「正常系」のコードだけを書くスタイルが広く使われています。

```ruby
def create
  user = User.find_by(email: params[:email])

  unless user
    flash.now[:alert] = "ユーザーが存在しません"
    return render :new, status: :unprocessable_entity
  end

  unless user.authenticate(params[:password])
    flash.now[:alert] = "パスワードが違います"
    return render :new, status: :unprocessable_entity
  end

  start_new_session_for(user)
  redirect_to root_path
end
```
## 早期リターンを使わない
例えば「18歳以上だけ入場できる」処理を書いてみます。
```ruby
def enter(age)
  if age < 18
    puts "入場できません"
    return
  end

  puts "入場できます"
end
```

## 早期リターンを使う
```ruby
def enter(age)
  if age >= 18
    puts "入場できます"
  else
    puts "入場できません"
  end
end
```
## ログインしていない
```ruby
def create
  return redirect_to login_path unless authenticated?

  # 日記を作成する
end
```
ログインしてないならログイン画面へ
してるなら日記作成へ
