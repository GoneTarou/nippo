## Rails 8.0の認証機能ジェネレータ　　

https://railsguides.jp/v8.0/security.html　　

Rails 8の認証を作るには次のコマンドを実行します。
```ruby
bin/rails generate authentication
```

このコマンド一つで、以下のファイルが自動生成されます

| カテゴリ | ファイルパス | 役割 |
|---------|------------|------|
| コントローラ | app/controllers/sessions_controller.rb | ログイン/ログアウト処理 |
| | app/controllers/passwords_controller.rb | パスワードリセット処理 |
| モデル | app/models/user.rb | ユーザーモデル |
| | app/models/session.rb | セッション管理モデル |
| | app/models/current.rb | Current.user の管理 |
| ビュー | app/views/sessions/new.html.erb | ログインフォーム |
| | app/views/passwords/new.html.erb | パスワードリセット申請フォーム |
| | app/views/passwords/edit.html.erb | パスワード再設定フォーム |
| メーラー | app/mailers/passwords_mailer.rb | パスワードリセットメール送信 |
| マイグレーション | db/migrate/xxx_create_users.rb | users テーブル作成 |
| | db/migrate/xxx_create_sessions.rb | sessions テーブル作成 |

Rails 8 では Cookie ベースのセッション管理から、データベースベースのセッション管理 に移行しています。
```ruby
# sessions テーブルにセッションを保存
@session = user.sessions.create!
cookies.signed.permanent[:session_token] = { value: @session.id, httponly: true }
```

| 項目 | Rails 7（従来） | Rails 8 |
|------|----------------|---------|
| セッション保存先 | Cookie（暗号化） | データベース（sessions テーブル） |
| セッション管理 | session[:user_id] | Session モデル |
| 複数デバイス対応 | 困難 | 簡単（sessions テーブルで管理） |
| ログアウト | Cookie 削除のみ | セッションレコードを削除 |
| セキュリティ | 基本的 | より高度（IP・User-Agent 記録） |

    
### 作成されるもの
usersテーブル
```ruby
create_table :users do |t|
  t.string :email_address, null: false
  t.string :password_digest, null: false

  t.timestamps
end
```
sessionsテーブル
```ruby
create_table :sessions do |t|
  t.references :user
  t.string :ip_address
  t.string :user_agent

  t.timestamps
end
```
ログイン処理  

ログイン画面でメールトパスワードを入力すると
```ruby
user = User.find_by(email_address: params[:email_address])

if user&.authenticate(params[:password])
```
Sessionモデル
```ruby
class Session < ApplicationRecord
  belongs_to :user
end
```
ログインすると
```ruby
Current.session = user.sessions.create!
```
すると、id,user_id,timestampが作られる。　

Rails8ではCurrent.userが使えるのでCurrent.user.nameでログインユーザーの名前が取得できる　　

### Rails 8標準認証の処理の流れ

┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃                  👤 ユーザー登録                      ┃
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

          📝 フォームに入力
          ├─ username
          ├─ email_address
          └─ password
                │
                ▼
          🔐 has_secure_password
          │   └─ BCryptでハッシュ化
                │
                ▼
          💾 データベースに保存
          │   ├─ username
          │   ├─ email_address
          │   └─ password_digest ← ハッシュ化された値
                │
                ▼
          ✅ 登録完了


┏━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┓
┃                  🔑 ログイン                      ┃ 
┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛

          📝 フォームに入力
          ├─ email_address
          └─ password
                │
                ▼
          🔍 User.find_by(email_address: ...)
          │   ├─ 見つかった → user オブジェクト
          │   └─ 見つからない → nil
                │
                ▼
          🔐 user&.authenticate(password)
          │   ├─ パスワード一致 → user オブジェクト
          │   └─ パスワード不一致 → false
                │
                ▼
          ✅ 認証成功
                │
                ▼
          🍪 セッションに保存
          │   └─ session[:user_id] = user.id
                │
                ▼
          👤 Current.user が利用可能
