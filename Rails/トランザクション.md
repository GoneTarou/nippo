## トランザクション

Railsのtransaction（トランザクション）とは、複数のデータベース操作を「全部成功」か「全部取り消し」にする仕組み  

例えば銀行振込を考えると、Aさんの口座 -10000円とBさんの口座 +10000円という2つの処理があります。  

Aさん -10000円でエラー発生　Bさん +10000円できないとなったら、お金が消えます。  

Aさん -10000円→Bさん +10000円→成功 → commit（確定）→失敗 → rollback（全部戻す）
```ruby
User.transaction do
  user.update!(balance: user.balance - 10000)
  receiver.update!(balance: receiver.balance + 10000)
end
```
このブロック内の処理が全部成功したら確定します。

途中で例外が起きると：raise ActiveRecord::Rollback  またはupdate!が失敗すると、rollbackされます。
### update と update! の違いが重要
update は失敗しても例外を出しません。  
つまりname更新失敗でも処理続行→email更新成功→commitになる可能性があります。
### モデル側でtransaction
```ruby
class OrderService
  def call
    Order.transaction do
      order.update!(status: "paid")
      Payment.create!(order: order)
      Stock.decrement!(product)
    end
  end
end
```

注文を支払い済みにする
決済記録を作る
在庫を減らす

全部成功したらOK。

## transactionで注意すること
1. 外部APIは入れない
悪い例
```ruby
User.transaction do
  user.update!
  
  Stripe.create_payment
end
```
DB transaction中に外部サービスを待つことになる。

例えばゲストユーザー本登録。
ゲストユーザー→正式ユーザーへ変更→Google UID紐付け→ゲストトークン削除
```ruby
User.transaction do
  user.update!(
    guest: false,
    google_uid: auth.uid,
    guest_token: nil
  )

  user.diaries.update_all(user_id: user.id)
end
```
途中失敗したら、ユーザーだけ更新された日記だけ移行されたという中途半端な状態を防げます。  
```ruby
User.transaction do
  user.update!
  post.update!
end
```
userの更新とpostの更新を1つの処理として扱う」という意味です。
user.update! が成功  post.update! も成功した場合だけ、両方保存します。
トランザクションを使わない場合、user更新は成功しても、post更新でエラーが発生した場合、usersテーブルは更新されてもpostsテーブルは更新されていないという状態になってしまう。

例えばユーザー削除。
```ruby
User.transaction do
  user.destroy!
  Diary.where(user: user).destroy_all
end
```
途中で日記削除に失敗したら、

ユーザーだけ消えた日記が残ったという状態になります。なのでtransactionで囲む。
### User.transaction の User は何？
User.transaction do の User は、「Userテーブルだけを対象にする」という意味ではありません。
単にUserモデル経由でデータベースのtransactionを開始するという意味です。
```ruby
User.transaction do
  user.update!
  post.update!
  comment.update!
end
```
でも全部同じtransactionになります。
## 覚え方としては：「複数のDB変更をセットにして、全部成功した時だけ保存したい時に使う」
